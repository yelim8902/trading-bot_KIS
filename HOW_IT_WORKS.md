# 트레이딩 봇 작동 원리

## 실행 방법

```bash
python bot.py              # 자동 스크리닝 모드 (거래량 상위 20개 분석)
python bot.py 005930       # 삼성전자 직접 지정
python bot.py 005930 000660  # 여러 종목 직접 지정
```

## 전체 흐름

```
[시작]
  │
  ├─ 직접 입력 모드 (python bot.py 005930)
  │     └─ 지정 종목 → 지표 계산 → Claude 판단 → 주문
  │
  └─ 자동 스크리닝 모드 (python bot.py)
        └─ 거래량 상위 20종목 수집
              └─ 1차 필터 (지표 조건)
                    └─ 2차 필터 (Claude AI 판단)
                          └─ 1위 종목 주문 실행

  30분마다 반복 | 장 외 시간엔 분석만, 주문 건너뜀
```

## 1단계: 후보 종목 수집

KIS API의 **거래량 상위 종목** 조회 (`volume-rank`)

| 필터 조건 | 설명 |
|-----------|------|
| 주가 5,000원 이상 | 초저가주 제외 |
| 주가 500,000원 이하 | 초고가주 제외 |
| 일 거래량 100,000주 이상 | 거래 없는 종목 제외 |

→ 상위 20개 종목 추출

---

## 2단계: 1차 필터 (코드 기반 빠른 스크리닝)

20개 종목 각각에 대해 일봉 데이터 30일치를 받아 4개 지표를 계산하고, **4가지 조건을 모두 통과한 종목만** 후보에 올림.

### 필터 조건

| 지표 | 조건 | 의미 |
|------|------|------|
| **RSI(14)** | 30 ≤ RSI ≤ 65 | 과매수(70+) 아닌 것. 아직 오를 여지 있음 |
| **MACD 히스토그램** | > 0 | 단기 상승 모멘텀 존재 |
| **볼린저밴드 위치** | < 75% | 상단 근접(저항선 근처) 아닌 것 |
| **거래량 비율** | ≥ 0.8배 | 20일 평균 대비 거래량이 80% 이상 (유동성 확보) |

### 각 지표 계산 방법

#### RSI (Relative Strength Index)
- 기간: 14일
- 계산: `RSI = 100 - (100 / (1 + 평균상승폭 / 평균하락폭))`
- 의미: 최근 14일 동안 얼마나 많이 올랐는지 0~100으로 표현

#### MACD (Moving Average Convergence Divergence)
- 계산: `MACD = EMA(12) - EMA(26)`
- 시그널: `EMA(9) of MACD`
- **히스토그램** = `MACD - 시그널` ← 이걸 사용
- 히스토그램 > 0: 단기 EMA가 장기 EMA보다 빠르게 상승 중 = 상승 모멘텀

#### 볼린저밴드
- 중단: 20일 이동평균
- 상단/하단: 중단 ± 2σ (표준편차의 2배)
- **밴드 위치** = `(현재가 - 하단) / (상단 - 하단)` → 0%(하단) ~ 100%(상단)
- 75% 미만: 상단 돌파까지 여유가 있음

#### 거래량 지표
- **거래량 비율** = `오늘 거래량 / 20일 평균 거래량`
  - 1.5 이상: 거래량 폭발 (강한 신호)
  - 0.8 미만: 거래량 부족 (신뢰도 낮음) → 1차 필터 탈락
- **거래량 추세** = `5일 평균 / 20일 평균` (1 초과 = 거래량 증가 추세)

## 3단계: 2차 필터 (Claude AI 판단)

1차 통과 종목들을 **Claude Opus 4.6** (adaptive thinking 활성화)에게 전달.

### Claude에게 주는 정보
- RSI, MACD (라인·시그널·히스토그램), 볼린저밴드 (상단·중단·하단·위치%), 거래량 (오늘/평균/비율/추세)

### Claude의 판단 기준
- 네 지표가 **같은 방향**이면 확신도 높게
- 거래량 폭발(1.5배+) + 가격 상승 = 강한 매수 신호
- 거래량 없는 상승(0.8배 미만) = 확신도 낮게
- 지표 엇갈릴 때 = HOLD 선호

### 출력 형식 (Pydantic 구조화)
```
action: BUY / SELL / HOLD
confidence: 0~100 (%)
target_price: 목표가 (원)
stop_loss: 손절가 (원)
reason: 판단 근거 (한국어)
```

→ **BUY + 확신도 70% 이상**인 종목만 최종 선발, 확신도 순 정렬

## 4단계: 주문 실행

### 리스크 관리 규칙
| 규칙 | 값 | 설명 |
|------|-----|------|
| 최소 확신도 | 70% | 미달 시 주문 안 냄 |
| 최대 포지션 | 현금의 30% | 한 종목에 몰빵 방지 |
| 주문 방식 | 시장가 | 즉시 체결 |
| 장 외 주문 | 차단 | 분석은 하되 주문은 09:00~15:30만 |

### 매수 수량 계산
```
budget = 주문 가능 현금 × 0.3
quantity = budget ÷ 현재가 (소수점 버림)
```

### 모의투자 vs 실전투자 자동 분기
- `KIS_URL`에 `vts` 포함 → 모의투자 tr_id 사용
- 그 외 → 실전투자 tr_id 사용

---

## 자동 루프

```
30분마다 실행
  → 장 운영 시간 (평일 09:00~15:30) 체크
  → 장 중: 분석 + 주문 실행
  → 장 외: 분석만 (주문 건너뜀)
  → 오류 발생해도 다음 사이클에 재시도
```
