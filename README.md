# 트레이딩 봇 (KIS API + Claude AI)

## 전체 동작 흐름

```
실행 (python bot.py)
    │
    ├─ 장 중 (09:00~15:30, 평일)
    │       ↓
    │   [스크리닝 사이클] ──────────────────────────────────┐
    │       │                                               │
    │   1단계: 후보 종목 수집                                │
    │       KIS API → 거래량 상위 20개 종목                  │
    │       │                                               │
    │   2단계: 1차 필터 (지표 기반)                          │
    │       RSI / MACD / 볼린저밴드 계산                     │
    │       조건 3개 모두 통과한 종목만 통과                  │
    │       │                                               │
    │   3단계: 2차 필터 (Claude AI 판단)                     │
    │       통과 종목만 Claude에게 전달                       │
    │       BUY + 확신도 70% 이상만 선택                     │
    │       │                                               │
    │   4단계: 주문 실행                                     │
    │       확신도 순 1위 종목 시장가 매수                    │
    │       30분 후 재실행 ──────────────────────────────────┘
    │
    └─ 장 외 시간
            분석은 동일하게 실행
            주문만 건너뜀 ("장 시작 후 주문 예정")
            30분마다 반복
```

---

## 1차 필터 판단 기준

거래량 상위 20개 종목에 대해 아래 **3가지 조건을 모두** 통과해야 2차 필터로 넘어갑니다.

### 조건 1: RSI (Relative Strength Index)

```
30 ≤ RSI(14) ≤ 65
```

| 범위 | 의미 | 판단 |
|------|------|------|
| RSI < 30 | 과매도 — 너무 많이 빠짐 | ❌ 제외 (반등 가능하지만 불안정) |
| 30 ~ 65 | 정상 범위 | ✅ 통과 |
| RSI > 65 | 과매수 근접 — 이미 많이 올랐음 | ❌ 제외 |

> RSI 70 초과가 보통 "과매수"지만, 여기선 65로 더 엄격하게 적용합니다.

---

### 조건 2: MACD 히스토그램

```
MACD 히스토그램 > 0
```

- MACD = 단기 EMA(12) − 장기 EMA(26)
- 히스토그램 = MACD − 시그널선(9)
- **히스토그램 > 0** = 단기 모멘텀이 장기보다 강함 = 상승 추세

| 값 | 의미 | 판단 |
|----|------|------|
| 히스토그램 > 0 | 상승 모멘텀 | ✅ 통과 |
| 히스토그램 ≤ 0 | 하락 또는 횡보 모멘텀 | ❌ 제외 |

---

### 조건 3: 볼린저밴드 위치 (bb_pct)

```
볼린저밴드 위치 < 75%
```

- `bb_pct = (현재가 − 하단) / (상단 − 하단)`
- 0% = 하단 / 50% = 중간 / 100% = 상단

| 값 | 의미 | 판단 |
|----|------|------|
| bb_pct < 20% | 하단 근접 — 반등 가능 | ✅ 통과 |
| 20% ~ 75% | 여유 있는 상태 | ✅ 통과 |
| bb_pct ≥ 75% | 상단 근접 — 추가 상승 여력 제한 | ❌ 제외 |

---

## 2차 필터: Claude AI 판단

1차 통과 종목의 지표 수치를 Claude Opus 4.6에게 전달합니다.

**Claude에게 주는 정보:**
- RSI, MACD, MACD 시그널, MACD 히스토그램
- 볼린저밴드 상단/중단/하단, 밴드 내 위치(%)

**Claude가 반환하는 값:**

```python
action:       "BUY" | "SELL" | "HOLD"
confidence:   0 ~ 100  # 확신도
target_price: int       # 목표가 (원)
stop_loss:    int       # 손절가 (원)
reason:       str       # 판단 근거 (한국어)
```

**주문 실행 조건:**
- `action == "BUY"` AND `confidence ≥ 70%`
- 둘 중 하나라도 미달이면 주문 없음

---

## 리스크 관리

| 항목 | 값 | 설명 |
|------|-----|------|
| 최소 확신도 | 70% | Claude 확신도 70% 미만이면 주문 취소 |
| 최대 포지션 | 30% | 보유 현금의 30%까지만 매수 |
| 주문 방식 | 시장가 | 즉시 체결 우선 |
| 매도 수량 | 1주 | (추후 보유 수량 연동 예정) |

## 실행 방법

```bash
# 자동 스크리닝 모드 (30분마다 거래량 상위 20종목 스캔)

# 특정 종목 직접 지정
python bot.py 005930

# 여러 종목 동시 분석
python bot.py 005930 000660

# 종료
Ctrl+C
```

## 설정값 (bot.py 상단에서 변경 가능)

```python
LOOP_INTERVAL     = 30    # 분: 스크리닝 반복 주기
MIN_CONFIDENCE    = 70    # %: Claude 최소 확신도
MAX_POSITION_RATIO = 0.3  # 보유 현금 대비 최대 매수 비율

# 1차 필터 기준 (screen_stocks 함수 내)
RSI 범위:  30 ~ 65
MACD 히스: > 0
BB 위치:   < 75%
```
